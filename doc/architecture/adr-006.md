# ADR 006: Tiered Lock Rewards

## 1. Overview

The `x/tieredrewards` module lets users **lock tokens into tiers** (e.g. 1-year, 5-year) to earn two types of rewards:

- **Base rewards** -- normal staking rewards from `x/distribution`, earned only while delegated to a bonded validator.
- **Bonus rewards** -- a fixed APY paid from an external rewards pool, earned only while delegated to a bonded validator while being **locked in a tier**. Bonus stops when the exit commitment elapses.

Locking always delegates to a validator in one step. Locked tokens are internal to the tier module -- they can only be moved via tier messages (delegate, redelegate, undelegate). The tier module account is the delegator in `x/staking`. Each position is all-or-nothing: the full amount is delegated to one validator.

Users can also **commit an existing delegation** to a tier without undelegating (MsgCommitDelegationToTier).

### Typical position lifecycle

```
Lock tokens + delegate to validator
        |
        v
   Earning rewards (base + bonus)
   [can: add tokens, redelegate, claim rewards]
        |
        v
   Trigger exit (start exit commitment timer, e.g. 1 year)
        |
        v
   Still earning rewards, but bonus is capped at exit commitment end
   [can: redelegate, claim rewards, cancel exit via ClearPosition]
        |
        v
   Exit commitment elapses (e.g. 1 year later) -- no more bonus
        |
        +---> Exit with delegation (instant, no unbonding)
        |     Transfer delegation back to owner on same validator.
        |     Supports partial exits. Position deleted if fully exited.
        |
        +---> Undelegate (starts staking unbonding)
              Wait for staking unbonding period (e.g. 21 days)
              Withdraw tokens -- position deleted
```

## 2. Position Lifecycle

### Entry

- **MsgLockTier(tier_id, amount, validator)** -- transfer tokens to module, delegate to validator, create position. Optional `trigger_exit_immediately` starts the exit clock at creation.
- **MsgCommitDelegationToTier(tier_id, validator, amount)** -- transfer an existing delegation to the tier module (same validator, no unbonding). Partial commit allowed. Optional `trigger_exit_immediately` starts the exit clock at creation.

Both require `amount >= tier.MinLockAmount`. Both create a delegated position immediately.

### While Active

- **MsgClaimTierRewards** -- claim pending base + bonus rewards for one or more positions owned by the signer in a single transaction. All position IDs must belong to the same owner (the message signer).
- **MsgAddToTierPosition** -- add tokens to an existing position (rejected if exiting).
- **MsgTierDelegate** -- delegate a position to a validator.
- **MsgTierRedelegate** -- move position to a different validator. Claims rewards first. Blocked after exit commitment elapses.
- **MsgClearPosition** -- cancel a triggered exit. Settles rewards first to prevent bonus manipulation. Resets exit timestamps.

### Exit (two paths after exit commitment)

1. **Trigger exit** -- `MsgTriggerExitFromTier`. Sets `ExitTriggeredAt` and `ExitUnlockAt`. Bonus continues accruing until `ExitUnlockAt`.

After exit commitment elapses (`block_time >= ExitUnlockAt`), the user chooses one of two paths:

2a. **Exit with delegation (instant)** -- `MsgExitTierWithDelegation(position_id, amount)`. Transfers the module's delegation back to the owner on the same validator (Unbond + Delegate, no unbonding queue). Claims rewards first. Supports partial exits: only the specified amount is transferred; position is reduced accordingly but position size needs to still be at least min lock amount. If the full amount is transferred, the position is deleted. Requires the validator to be bonded.

2b. **Undelegate + Withdraw (21-day wait)** -- `MsgTierUndelegate` claims rewards and starts the staking unbonding period. After unbonding completes, `MsgWithdrawFromTier` transfers `Amount` to the owner and deletes the position.

**Cancel exit** -- `MsgClearPosition` can cancel exit at any point during or after the exit commitment, as long as the position is delegated. Settles rewards first, then resets exit timestamps. This allows the user to re-lock the position and resume earning bonus rewards.

### CloseOnly Tiers

When a tier is marked `CloseOnly` by governance, these messages are **blocked**: LockTier, CommitDelegation, AddToPosition, TierDelegate, TierRedelegate, ClearPosition. **Allowed**: TriggerExit, TierUndelegate, WithdrawFromTier, ClaimTierRewards, ExitTierWithDelegation. This lets governance sunset a tier while existing positions close out.

## 3. State Model

### Tier

```go
type Tier struct {
    Id            uint32          // e.g. 1, 2, 3
    ExitDuration  time.Duration   // wait time after triggering exit before undelegate is allowed
    BonusApy      sdk.Dec         // fixed bonus APY (e.g. 0.04 = 4%/year); capped at 1.0
    MinLockAmount math.Int        // minimum lock amount for new positions
    CloseOnly     bool            // when true, no new positions allowed (see Â§2)
}
```

Tiers are managed by governance (`MsgAddTier`, `MsgUpdateTier`, `MsgDeleteTier`), stored as separate keyed entries.

### Params

```go
type Params struct {
    TargetBaseRewardsRate sdk.Dec // used by BeginBlocker to top up base rewards from pool; capped at 1.0
}
```

### Position

```go
type Position struct {
    Id                  uint64
    Owner               string       // bech32 account address
    TierId              uint32
    Amount              math.Int     // For delegated positions, Amount is always zero â€” the live
                                     // token value is derived from validator.TokensFromShares(DelegatedShares).
                                     // For undelegated positions, Amount holds the actual token value.
    CreatedAtHeight     uint64
    CreatedAtTime       time.Time
    ExitTriggeredAt     time.Time    // zero = not exiting
    ExitUnlockAt        time.Time    // ExitTriggeredAt + tier.ExitDuration
    Validator           string       // validator operator address; empty = not delegated
    DelegatedShares     sdk.Dec      // staking shares for this delegation
    BaseRewardsPerShare sdk.DecCoins // snapshot of validator's cumulative ratio at last claim
    LastBonusAccrual    time.Time    // upper bound of last bonus accrual period
    LastEventSeq        uint64       // sequence of last processed validator event
    LastKnownBonded     bool         // persisted bonded state for lazy bonus replay
}
```

**Validation rules:** When delegated (`Validator != ""`): `DelegatedShares` must be positive. When not delegated: `DelegatedShares`, `BaseRewardsPerShare`, and `LastBonusAccrual` must be zero/empty.

### PositionResponse

`PositionResponse` is a query-only flat struct mirroring Position fields but with `Amount` = computed token value (`TokensFromShares` for delegated positions, stored `Amount` for undelegated positions). All query responses (`TierPosition`, `TierPositionsByOwner`, `AllTierPositions`) use `PositionResponse`. Validator rate is cached per query for list endpoints.

## 4. Messages

### Reference Table

| Message | Description | Key Validations |

| **MsgLockTier** | Lock tokens + delegate to validator. Optional `trigger_exit_immediately`. | amount >= MinLockAmount; validator bonded; tier not close-only |

| **MsgTierDelegate** | Delegate an undelegated position. | Not already delegated; validator bonded; exit not elapsed; tier not close-only; Amount > 0 |

| **MsgWithdrawFromTier** | Withdraw tokens + delete position. | Exit triggered; exit elapsed; not delegated; no pending unbonding |
| **MsgClaimTierRewards** | Claim base + bonus rewards for one or more positions. All positions must belong to the signer. | Owner match on all positions; position_ids non-empty, no duplicates, max 500; returns zero per position if not delegated |
| **MsgExitTierWithDelegation** | Transfer delegation back to owner (no unbonding). Supports partial exits. | **MsgUpdateParams** | Update module params. | Authority (gov) |
| **MsgAddTier** / **MsgUpdateTier** / **MsgDeleteTier** | Manage tiers. DeleteTier fails if positions exist. | Authority (gov) |

### MsgLockTier Flow

```
-> Validate tier exists, not close-only, amount >= MinLockAmount, validator bonded
-> Bank.SendCoinsFromAccountToModule(owner, tieredrewards, amount)
-> Update ValidatorRewardRatio for validator
-> Staking.Delegate(tier_module_account, validator, amount) -> newShares
-> Create Position: Amount = TokensFromShares(newShares), DelegatedShares = newShares,
   BaseRewardsPerShare = currentRatio, LastBonusAccrual = block_time
-> If trigger_exit_immediately: set ExitTriggeredAt, ExitUnlockAt
```

### MsgCommitDelegationToTier Flow

```
-> Validate amount <= user's delegation to validator, amount >= MinLockAmount
-> Update ValidatorRewardRatio for validator
-> TransferDelegation(owner -> tier_module_account, validator, shares)
   (internally: Unbond user's shares, re-Delegate from module account; no unbonding period)
-> Create Position with transferred shares
```

### MsgClaimTierRewards Flow

```
-> Validate: owner match, position_ids non-empty, no duplicates
-> For each position: get position, validate ownership
-> Skip undelegated positions (return zero rewards for those)

-> Group delegated positions by validator, then by tier within each validator
-> For each validator group:
     Phase 1 (Base â€” batched per validator):
     -> Withdraw distribution rewards for validator (deduplicated per block via transient store)
     -> ValidatorRewardRatio[V] += withdrawn / total_module_shares_on_V
     -> For each position: reward = DelegatedShares * (currentRatio - BaseRewardsPerShare)
     -> Accumulate total base rewards; emit per-position EventBaseRewardsClaimed
     -> Single batched bank send for all base rewards in the validator group

     Phase 2 (Bonus â€” per position):
     -> Walk pending events since pos.LastEventSeq
     -> For each bonded segment [segmentStart, eventTime):
        bonus += shares Ă— event.TokensPerShare Ă— apy Ă— duration / SecondsPerYear
     -> Update bonded state based on event type (UNBONDâ†’false, BONDâ†’true, SLASHâ†’unchanged)
     -> Decrement event reference count (garbage-collect if zero)
     -> Final segment: if bonded && val.IsBonded():
        bonus += shares Ă— currentRate Ă— apy Ă— duration
     -> Advance LastBonusAccrual, LastEventSeq, LastKnownBonded
     -> Accumulate total bonus; emit per-position EventBonusRewardsClaimed
     -> If pool balance < total bonus: fail atomically (user retries later)
     -> Single batched bank send for all bonus rewards in the tier group

-> Persist all updated positions
-> Emit single aggregated EventTierRewardsClaimed with position_ids and totals
-> Return aggregated base_rewards, bonus_rewards, position_ids
```

Validator and tier lookups are cached across positions to reduce gas. Bank sends
are batched (one per validator group for base, one per validator+tier group for
bonus) instead of per-position, significantly reducing gas for multi-position claims.

**Reward settlement before mutations:** MsgTierUndelegate, MsgTierRedelegate, MsgAddToTierPosition, MsgClearPosition, and MsgExitTierWithDelegation all claim rewards before modifying the position. This prevents double-counting and ensures bonus is capped at `ExitUnlockAt` before the cap is removed.

**Amount reconciliation:** When delegating, `Amount` is set to `validator.TokensFromShares(shares).TruncateInt()` rather than the raw input amount. This accounts for share-to-token rounding (may differ by up to 1 unit).

### MsgExitTierWithDelegation Flow

```
-> Validate: owner match, delegated, exit triggered, exit elapsed, amount > 0, amount <= position amount, no active redelegation
-> Claim rewards for position (settle base + bonus)
-> If amount == position amount (full exit): use all DelegatedShares
   Else (partial): convert amount to shares via ValidateUnbondAmount
-> transferDelegationFromTier: Unbond(moduleAddr, valAddr, shares) -> returnAmount
   Re-fetch validator, Delegate(ownerAddr, returnAmount, validator) -- instant, no unbonding
-> If full exit: delete position (all indexes cleaned up)
   Else: reduce DelegatedShares, reconcile Amount from remaining shares
     reject if remaining Amount < tier MinLockAmount (post-transfer check on actual amount)
     save position
-> Emit EventExitTierWithDelegation(position_id, tier_id, owner, validator, amount, full_exit)
```

## 5. Reward Mechanics

### Base Rewards (Cumulative Rewards-Per-Share)

The tier module has one delegation per validator in `x/staking` (all positions on that validator share it). To attribute rewards to individual positions:

1. **`ValidatorRewardRatio[V]`** -- cumulative rewards-per-share. Each time distribution rewards are withdrawn for V: `ratio += withdrawn / total_module_shares_on_V`.
2. **`position.BaseRewardsPerShare`** -- snapshot at creation or last claim.
3. **On claim:** `reward = DelegatedShares * (currentRatio - BaseRewardsPerShare)`. Then update snapshot.

**Deduplication:** Withdrawals are deduplicated per validator per block via transient store.

**When ratio is updated:** Before any operation that changes the module's shares for a validator (delegate, undelegate, redelegate, slash). This ensures the ratio reflects rewards earned under the pre-mutation share count.

### Bonus Rewards (Lazy Validator Events)

Bonus rewards are computed lazily using a validator event log rather than eagerly settling all positions on every validator state change.

**Event recording:** Staking hooks record `ValidatorEvent` entries (types: `SLASH`, `UNBOND`, `BOND`) with `TokensPerShare` snapshots at O(1) cost. Each event stores a `ReferenceCount` equal to the number of delegated positions on that validator at event time (from `PositionCountByValidator`).

**Claim-time replay:** At claim time, `processEventsAndClaimBonus` walks pending events for each position (from `pos.LastEventSeq` forward). For each bonded segment between events, bonus is computed using the snapshot rate at the segment boundary. Events are reference-counted; the count is decremented when a position processes the event, and the event is garbage-collected when the count reaches zero.

**Bonded state tracking:** `LastKnownBonded` tracks whether the validator was bonded after the last event replay. Event types update this state: `UNBOND` sets it to false, `BOND` sets it to true, `SLASH` leaves it unchanged.

**Formula per segment:** `shares * tokensPerShare * bonusApy * durationSeconds / SecondsPerYear`

**Accrual cap:** Bonus accrual is capped at `ExitUnlockAt` when exiting. No bonus after exit commitment elapses.

**Insufficient pool handling:**
- **User-driven paths** (ClaimTierRewards, AddToPosition, Undelegate, Redelegate, ClearPosition): fail atomically. User retries after pool is replenished.

### BeginBlocker: Base Rewards Top-Up

If `TargetBaseRewardsRate > 0`, the BeginBlocker computes the target per-block staker reward, compares it to the fee collector balance (after community tax), and transfers any shortfall from the rewards pool to the distribution module, allocated pro-rata by validator power.

## 6. Hooks & Slashing

| Hook | Behavior |
|------|----------|
| **AfterValidatorBeginUnbonding** | Records `UNBOND` event with `TokensPerShare` snapshot at O(1). No position iteration. |
| **AfterValidatorBonded** | Records `BOND` event with `TokensPerShare` snapshot at O(1). No position iteration. |
| **AfterValidatorRemoved** | If no leftover events (`hasValidatorEvents` returns false): clears `ValidatorRewardRatio` and `ValidatorEventSeq`. If leftover events exist: skips cleanup (ratio and seq needed for future claims). |
| **BeforeValidatorSlashed** | Records `SLASH` event at O(1) with pre-slash `TokensPerShare` snapshot. No position iteration. No `updatePoolDelegationInfo` (distribution handles slash accounting via `ValidatorSlashEvent`). |
| **AfterUnbondingDelegationSlashed** | Reduce position `Amount` by slash amount. |
| **AfterUnbondingRedelegationSlashed** | Reduce position `Amount` by slash amount. |
| **AfterRedelegationSlashed** | Claims pending base and bonus rewards (via `claimBaseRewards` + `processEventsAndClaimBonus`) before reducing `Amount` and `DelegatedShares` (shares burnt). If bonus pool insufficient, logs error and continues. |
| **AfterUnbondingCompleted** | Clean up unbonding-to-position mapping. |
| **AfterRedelegationCompleted** | Clean up redelegation-to-position mapping. |

## 7. Governance Voting

The tier module account is the delegator in staking, so tier lockers have no voting power by default. A custom tally function (via `WithCustomCalculateVoteResultsAndVotingPowerFn`) adds tier voting power:

1. For each voter, compute standard staking power (iterate personal delegations).
2. Call `GetPositionsByOwner(ctx, voter)`. For each delegated position whose validator is in the bonded validators map: `tierPower = DelegatedShares * BondedTokens / DelegatorShares`.
3. Add `DelegatedShares` to the validator's `DelegatorDeductions` to prevent double-counting (the tier module account's delegation shares must not also count in the validator's second-pass residual power).

Positions on non-bonded validators yield zero. Undelegated positions yield zero. Tier-locked tokens are included in `TotalBondedTokens` (they are bonded through the tier module), so quorum is not inflated.

## 8. Edge Cases

| Case | Behavior |
|------|----------|
| Pool empty (user-driven) | Message fails atomically. No state change. Retry after pool replenished. |
| Slashing | `Amount` recalculated from shares (pre-slash). `DelegatedShares` unchanged (except redelegation slashes). Bonus accrues on updated token value. |
| Validator jailed/unbonding | Position stays valid. No base rewards while inactive. No bonus while not bonded. User can redelegate or undelegate (if exit elapsed). ExitTierWithDelegation fails.|
| ClearPosition after exit elapsed | Requires position is delegated and not unbonding. Settles rewards first. Resets `LastBonusAccrual` to block_time if delegated. |
| Add to position when exiting | Rejected. Use ClearPosition first to cancel exit. |
| Partial ExitTierWithDelegation below MinLockAmount | Rejected. Remaining position amount must meet the tier's MinLockAmount. Full exit is always allowed regardless of MinLockAmount. |
| Zero-amount position (post-slash) | Valid state. No rewards. Can still withdraw. |
| Redelegate from unbonded src validator | SDK returns `unbondingId=0`, no redelegation mapping created. Second redelegate succeeds immediately. |
| Redelegate to non-bonded validator | Rejected with `ErrValidatorNotBonded`. |

## 9. Security Invariants

- Only governance can manage tiers and params.
- Rewards accrue only when delegated to a bonded validator.
- No undelegation until exit commitment has elapsed.
- `MsgClearPosition` settles rewards before clearing exit to prevent bonus manipulation past `ExitUnlockAt`.
- Pool balance is never exceeded. User paths fail atomically; hook paths forfeit.
- Only the tier module can delegate/undelegate/redelegate tier-locked tokens.
- CloseOnly tiers block new positions while allowing exits.
- Bonus cannot be double-claimed: `LastEventSeq` prevents event replay, `LastBonusAccrual` prevents segment replay, `LastKnownBonded` prevents unbonded gap overpay.
- All delegation paths reject non-bonded validators (delegate, redelegate, transferDelegationToTier).
- Genesis validation cross-checks event `ReferenceCount` against position `LastEventSeq` values.
