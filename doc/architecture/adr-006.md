# ADR 006: Tiered Lock Rewards

## 1. Overview

The `x/tieredrewards` module lets users **lock tokens into tiers** (e.g. 1-year, 5-year) to earn two types of rewards:

- **Base rewards** -- normal staking rewards from `x/distribution`, earned only while delegated to a bonded validator.
- **Bonus rewards** -- a fixed APY paid from an external rewards pool, earned only while delegated to a bonded validator while being **locked in a tier**. Bonus stops when the exit commitment elapses.

Locking always delegates to a validator in one step. Locked tokens are internal to the tier module -- they can only be moved via tier messages (delegate, redelegate, undelegate). The tier module account is the delegator in `x/staking`. Each position is all-or-nothing: the full amount is delegated to one validator.

Users can also **commit an existing delegation** to a tier without undelegating (MsgCommitDelegationToTier).

### Typical position lifecycle

```
Lock tokens + delegate to validator
        |
        v
   Earning rewards (base + bonus)
   [can: add tokens, redelegate, claim rewards]
        |
        v
   Trigger exit (start exit commitment timer, e.g. 1 year)
        |
        v
   Still earning rewards, but bonus is capped at exit commitment end
   [can: redelegate, claim rewards, cancel exit via ClearPosition]
        |
        v
   Exit commitment elapses (e.g. 1 year later) -- no more bonus
        |
        v
   Undelegate (claims all remaining rewards, starts staking unbonding)
        |
        v
   Wait for staking unbonding period (e.g. 21 days)
        |
        v
   Withdraw tokens -- position deleted
```

## 2. Position Lifecycle

### Entry

- **MsgLockTier(tier_id, amount, validator)** -- transfer tokens to module, delegate to validator, create position. Optional `trigger_exit_immediately` starts the exit clock at creation.
- **MsgCommitDelegationToTier(tier_id, validator, amount)** -- transfer an existing delegation to the tier module (same validator, no unbonding). Partial commit allowed. Optional `trigger_exit_immediately` starts the exit clock at creation.

Both require `amount >= tier.MinLockAmount`. Both create a delegated position immediately.

### While Active

- **MsgClaimTierRewards** -- claim pending base + bonus rewards.
- **MsgAddToTierPosition** -- add tokens to an existing position (rejected if exiting).
- **MsgTierDelegate** -- delegate a position to a validator.
- **MsgTierRedelegate** -- move position to a different validator. Claims rewards first. Blocked after exit commitment elapses.
- **MsgClearPosition** -- cancel a triggered exit. Settles rewards first to prevent bonus manipulation. Resets exit timestamps.

### Exit (three phases)

1. **Trigger exit** -- `MsgTriggerExitFromTier`. Sets `ExitTriggeredAt` and `ExitUnlockAt`. Bonus continues accruing until `ExitUnlockAt`.
2. **Undelegate** -- `MsgTierUndelegate`. Only allowed after exit commitment elapses (`block_time >= ExitUnlockAt`). Claims all pending rewards before undelegating. Clears delegation state immediately. Records unbonding mapping for slash handling.
3. **Withdraw** -- `MsgWithdrawFromTier`. Only after unbonding completes. Transfers `Amount` to owner, deletes position. No rewards claimed at this step (already settled at undelegate).

### CloseOnly Tiers

When a tier is marked `CloseOnly` by governance, these messages are **blocked**: LockTier, CommitDelegation, AddToPosition, TierDelegate, TierRedelegate, ClearPosition. **Allowed**: TriggerExit, TierUndelegate, WithdrawFromTier, ClaimTierRewards. This lets governance sunset a tier while existing positions close out.

## 3. State Model

### Tier

```go
type Tier struct {
    Id            uint32          // e.g. 1, 2, 3
    ExitDuration  time.Duration   // wait time after triggering exit before undelegate is allowed
    BonusApy      sdk.Dec         // fixed bonus APY (e.g. 0.04 = 4%/year); capped at 1.0
    MinLockAmount math.Int        // minimum lock amount for new positions
    CloseOnly     bool            // when true, no new positions allowed (see §2)
}
```

Tiers are managed by governance (`MsgAddTier`, `MsgUpdateTier`, `MsgDeleteTier`), stored as separate keyed entries.

### Params

```go
type Params struct {
    TargetBaseRewardsRate sdk.Dec // used by BeginBlocker to top up base rewards from pool; capped at 1.0
}
```

### Position

```go
type Position struct {
    Id                  uint64
    Owner               string       // bech32 account address
    TierId              uint32
    Amount              math.Int     // locked tokens (bond denom)
    CreatedAtHeight     uint64
    CreatedAtTime       time.Time
    ExitTriggeredAt     time.Time    // zero = not exiting
    ExitUnlockAt        time.Time    // ExitTriggeredAt + tier.ExitDuration
    Validator           string       // validator operator address; empty = not delegated
    DelegatedShares     sdk.Dec      // staking shares for this delegation
    BaseRewardsPerShare sdk.DecCoins // snapshot of validator's cumulative ratio at last claim
    LastBonusAccrual    time.Time    // upper bound of last bonus accrual period
}
```

**Validation rules:** When delegated (`Validator != ""`): `DelegatedShares` must be positive. When not delegated: `DelegatedShares`, `BaseRewardsPerShare`, and `LastBonusAccrual` must be zero/empty.

### Store Layout

| Key | Type | Purpose |
|-----|------|---------|
| Positions | position_id -> Position | Primary store |
| PositionsByOwner | (owner, position_id) -> {} | List by owner |
| PositionsByTier | (tier_id, position_id) -> {} | List by tier |
| PositionsByValidator | (validator, position_id) -> {} | List by validator |
| PositionCountByTier | tier_id -> uint64 | Count per tier (for delete validation) |
| NextPositionId | uint64 | Auto-incrementing sequence |
| ValidatorRewardRatio | validator -> DecCoins | Cumulative rewards-per-share (§5) |
| UnbondingIdToPositionId | unbonding_id -> position_id | Slash handling |
| UnbondingIdsByPosition | position_id -> unbonding_id | Reverse mapping |
| RedelegationIdToPositionId | redelegation_id -> position_id | Slash handling |
| RedelegationIdsByPosition | position_id -> redelegation_id | Reverse mapping |

## 4. Messages

### Reference Table

| Message | Description | Key Validations |
|---------|-------------|-----------------|
| **MsgLockTier** | Lock tokens + delegate to validator. Optional `trigger_exit_immediately`. | amount >= MinLockAmount; validator bonded; tier not close-only |
| **MsgCommitDelegationToTier** | Transfer existing delegation to tier (no unbonding). Partial allowed. | amount <= user's delegation; amount >= MinLockAmount; tier not close-only |
| **MsgAddToTierPosition** | Add tokens to existing position. If delegated, claims rewards first and delegates new amount. | Not exiting; tier not close-only |
| **MsgTierDelegate** | Delegate an undelegated position. | Not already delegated; validator bonded; exit not elapsed; tier not close-only; Amount > 0 |
| **MsgTierRedelegate** | Move delegation to another validator. Claims rewards first. | Exit not elapsed; tier not close-only; Amount > 0; dst != src |
| **MsgTierUndelegate** | Undelegate after exit commitment. Claims rewards first. Clears delegation state immediately. | Exit triggered; exit elapsed; delegated |
| **MsgTriggerExitFromTier** | Start exit commitment. | Not already exiting |
| **MsgClearPosition** | Cancel exit. Settles rewards first. If delegated, resets `LastBonusAccrual` to block_time. No-op if not exiting. | Tier not close-only; if exit elapsed: must be delegated and not unbonding |
| **MsgWithdrawFromTier** | Withdraw tokens + delete position. | Exit triggered; exit elapsed; not delegated; no pending unbonding |
| **MsgClaimTierRewards** | Claim base + bonus rewards. | Owner match; returns zero if not delegated |
| **MsgUpdateParams** | Update module params. | Authority (gov) |
| **MsgAddTier** / **MsgUpdateTier** / **MsgDeleteTier** | Manage tiers. DeleteTier fails if positions exist. | Authority (gov) |

### MsgLockTier Flow

```
-> Validate tier exists, not close-only, amount >= MinLockAmount, validator bonded
-> Bank.SendCoinsFromAccountToModule(owner, tieredrewards, amount)
-> Update ValidatorRewardRatio for validator
-> Staking.Delegate(tier_module_account, validator, amount) -> newShares
-> Create Position: Amount = TokensFromShares(newShares), DelegatedShares = newShares,
   BaseRewardsPerShare = currentRatio, LastBonusAccrual = block_time
-> If trigger_exit_immediately: set ExitTriggeredAt, ExitUnlockAt
```

### MsgCommitDelegationToTier Flow

```
-> Validate amount <= user's delegation to validator, amount >= MinLockAmount
-> Update ValidatorRewardRatio for validator
-> TransferDelegation(owner -> tier_module_account, validator, shares)
   (internally: Unbond user's shares, re-Delegate from module account; no unbonding period)
-> Create Position with transferred shares
```

### MsgClaimTierRewards Flow

```
-> If not delegated: return zero rewards

Phase 1 (Base):
-> Withdraw distribution rewards for validator (deduplicated per block via transient store)
-> ValidatorRewardRatio[V] += withdrawn / total_module_shares_on_V
-> reward = DelegatedShares * (currentRatio - position.BaseRewardsPerShare)
-> Send truncated reward to owner; set BaseRewardsPerShare = currentRatio

Phase 2 (Bonus):
-> accrual_end = min(block_time, ExitUnlockAt) if exiting, else block_time
-> duration_seconds = accrual_end - LastBonusAccrual (skip if <= 0)
-> tokens = validator.TokensFromShares(DelegatedShares)
-> bonus = tokens * BonusApy * duration_seconds / SecondsPerYear (31,557,600)
-> If pool balance < bonus: fail atomically (user retries later)
-> Send bonus from rewards pool to owner; set LastBonusAccrual = accrual_end
```

**Reward settlement before mutations:** MsgTierUndelegate, MsgTierRedelegate, MsgAddToTierPosition, and MsgClearPosition all claim rewards before modifying the position. This prevents double-counting and ensures bonus is capped at `ExitUnlockAt` before the cap is removed.

**Amount reconciliation:** When delegating, `Amount` is set to `validator.TokensFromShares(shares).TruncateInt()` rather than the raw input amount. This accounts for share-to-token rounding (may differ by up to 1 unit).

## 5. Reward Mechanics

### Base Rewards (Cumulative Rewards-Per-Share)

The tier module has one delegation per validator in `x/staking` (all positions on that validator share it). To attribute rewards to individual positions:

1. **`ValidatorRewardRatio[V]`** -- cumulative rewards-per-share. Each time distribution rewards are withdrawn for V: `ratio += withdrawn / total_module_shares_on_V`.
2. **`position.BaseRewardsPerShare`** -- snapshot at creation or last claim.
3. **On claim:** `reward = DelegatedShares * (currentRatio - BaseRewardsPerShare)`. Then update snapshot.

**Deduplication:** Withdrawals are deduplicated per validator per block via transient store.

**When ratio is updated:** Before any operation that changes the module's shares for a validator (delegate, undelegate, redelegate, slash). This ensures the ratio reflects rewards earned under the pre-mutation share count.

### Bonus Rewards (Fixed APY)

**Formula:** `bonus = TokensFromShares(DelegatedShares) * BonusApy * duration_seconds / SecondsPerYear`

- Uses the token value of shares (via `validator.TokensFromShares`), so slashing is automatically reflected.
- Accrual is capped at `ExitUnlockAt` when exiting. No bonus after exit commitment elapses.
- No bonus when validator is not bonded (except during `forceAccrue` hooks).

**Insufficient pool handling:**
- **User-driven paths** (ClaimTierRewards, AddToPosition, Undelegate, Redelegate, ClearPosition): fail atomically. User retries after pool is replenished.
- **Hook-driven paths** (AfterValidatorBeginUnbonding, BeforeValidatorSlashed): advance `LastBonusAccrual` regardless. Unpaid bonus is permanently forfeited. This ensures validator state transitions are never blocked by the tier module.

### BeginBlocker: Base Rewards Top-Up

If `TargetBaseRewardsRate > 0`, the BeginBlocker computes the target per-block staker reward, compares it to the fee collector balance (after community tax), and transfers any shortfall from the rewards pool to the distribution module, allocated pro-rata by validator power.

## 6. Hooks & Slashing

| Hook | Behavior |
|------|----------|
| **AfterValidatorBeginUnbonding** | Settle rewards for all positions on this validator with `forceAccrue=true` (SDK already changed status to Unbonding before firing). |
| **AfterValidatorBonded** | Reset `LastBonusAccrual` to block_time for all positions (bonus accrues only from when validator is bonded). |
| **AfterValidatorRemoved** | Clear `ValidatorRewardRatio` for this validator. |
| **BeforeValidatorSlashed** | Settle rewards with `forceAccrue=false` (bonus only if still bonded). Update `updatePoolDelegationInfo` to adjust distribution's `DelegatorStartingInfo.Stake` by `(1 - fraction)`. Recalculate each position's `Amount` as `TokensFromShares(DelegatedShares) * (1 - fraction)` (hook fires before SDK applies slash). `DelegatedShares` is not changed. |
| **AfterUnbondingDelegationSlashed** | Reduce position `Amount` by slash amount. |
| **AfterUnbondingRedelegationSlashed** | Reduce position `Amount` by slash amount. |
| **AfterRedelegationSlashed** | Reduce position `Amount` by slash amount and update `DelegatedShares` (shares burnt). |
| **AfterUnbondingCompleted** | Clean up unbonding-to-position mapping. |
| **AfterRedelegationCompleted** | Clean up redelegation-to-position mapping. |

## 7. Governance Voting

The tier module account is the delegator in staking, so tier lockers have no voting power by default. A custom tally function (via `WithCustomCalculateVoteResultsAndVotingPowerFn`) adds tier voting power:

1. For each voter, compute standard staking power (iterate personal delegations).
2. Call `GetPositionsByOwner(ctx, voter)`. For each delegated position whose validator is in the bonded validators map: `tierPower = DelegatedShares * BondedTokens / DelegatorShares`.
3. Add `DelegatedShares` to the validator's `DelegatorDeductions` to prevent double-counting (the tier module account's delegation shares must not also count in the validator's second-pass residual power).

Positions on non-bonded validators yield zero. Undelegated positions yield zero. Tier-locked tokens are included in `TotalBondedTokens` (they are bonded through the tier module), so quorum is not inflated.

## 8. Edge Cases

| Case | Behavior |
|------|----------|
| Pool empty (user-driven) | Message fails atomically. No state change. Retry after pool replenished. |
| Pool empty (hook-driven) | Checkpoint advanced, unpaid bonus forfeited permanently. Validator state transition proceeds. |
| Slashing | `Amount` recalculated from shares (pre-slash). `DelegatedShares` unchanged (except redelegation slashes). Bonus accrues on updated token value. |
| Validator jailed/unbonding | Position stays valid. No base rewards while inactive. No bonus while not bonded. User can redelegate or undelegate (if exit elapsed). |
| ClearPosition after exit elapsed | Requires position is delegated and not unbonding. Settles rewards first. Resets `LastBonusAccrual` to block_time if delegated. |
| Add to position when exiting | Rejected. Use ClearPosition first to cancel exit. |
| Zero-amount position (post-slash) | Valid state. No rewards. Can still withdraw. |

---

## 9. Security Invariants

- Only governance can manage tiers and params.
- Rewards accrue only when delegated to a bonded validator.
- No undelegation until exit commitment has elapsed.
- `MsgClearPosition` settles rewards before clearing exit to prevent bonus manipulation past `ExitUnlockAt`.
- Pool balance is never exceeded. User paths fail atomically; hook paths forfeit.
- Only the tier module can delegate/undelegate/redelegate tier-locked tokens.
- CloseOnly tiers block new positions while allowing exits.
