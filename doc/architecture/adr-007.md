# ADR 007: Inflation Control Module

## 1. Overview

The `x/inflation` module controls token supply and inflation decay for the chain. It provides three capabilities:

1. **Maximum supply cap** -- halts the chain if circulating supply exceeds a configured limit.
2. **Burned address tracking** -- excludes tokens held at designated burn addresses from the circulating supply calculation.
3. **Exponential inflation decay** -- gradually reduces the inflation rate over time by applying a monthly decay factor to the base inflation from the Mint module.

### How it works

```
Each block:
  1. Mint module mints new tokens using the decayed inflation rate
  2. Inflation module checks: circulating supply <= max supply
     (circulating = total supply - burned address balances)
     If exceeded: chain halts.

Inflation rate per block:
  inflation = base_rate × (1 - decay_rate) ^ months_elapsed
  where months_elapsed = (current_height - decay_epoch) / blocks_per_month
```

The module does not mint tokens itself. It provides a `DeflationCalculationFn` to the Mint module that modifies the inflation rate, and runs a BeginBlocker that enforces the supply cap.

---

## 2. Parameters

```go
type Params struct {
    MaxSupply       math.Int       // 0 = unlimited; positive = hard cap on circulating supply
    BurnedAddresses []string       // bech32 addresses whose balances are excluded from circulating supply
    DecayRate       math.LegacyDec // monthly decay rate [0, 1]; 0 = no decay; 1 = no minting
}
```

Updated via governance (`MsgUpdateParams`). All three fields must be supplied.

**Validation:** MaxSupply >= 0. BurnedAddresses must be valid bech32 and unique. DecayRate in [0, 1].

### Decay Epoch

The block height at which decay begins is stored in module state (`decay_epoch_start`), not in Params. It is set by the upgrade handler (e.g., v7.0.0 sets it to the upgrade activation height) or via genesis. It is not changed by `MsgUpdateParams`.

---

## 3. Supply Cap (BeginBlocker)

Runs every block **after** the Mint module (which mints new tokens):

```
circulating_supply = bank.TotalSupply(bond_denom) - sum(bank.Balance(addr, bond_denom) for addr in BurnedAddresses)
if MaxSupply > 0 and circulating_supply > MaxSupply:
    panic("the total supply has exceeded the maximum supply")  // chain halts
```

The chain halt is intentional -- it prevents further minting beyond the cap. Recovery requires a governance upgrade or parameter change.

---

## 4. Inflation Decay (Mint Integration)

The module provides `DeflationCalculationFn()` which the Mint module calls every block to determine the inflation rate.

### Formula

```
inflation_rate = base_rate × (1 - decay_rate) ^ months_elapsed
```

Where:
- `base_rate` = default Cosmos SDK inflation calculation (considers bonded ratio, min/max inflation, rate change)
- `decay_rate` = `Params.DecayRate` (monthly decay factor)
- `months_elapsed` = `(current_height - decay_epoch_start) / (blocks_per_year / 12)` (continuous decimal)

### When decay is NOT applied (base rate returned as-is):
- `DecayRate == 0`
- `decay_epoch_start` not set
- `current_height < decay_epoch_start`

### Computation optimization

The formula is decomposed to avoid underflow with large exponents:

```
(1 - d)^months = (1 - d)^n × ((1 - d)^(1/blocks_per_month))^r
```

Where `n` = integer months, `r` = remainder blocks. The per-block factor `(1-d)^(1/blocks_per_month)` is cached to avoid expensive `ApproxRoot` calls every block.

## 7. Genesis

```go
type GenesisState struct {
    Params          Params
    DecayEpochStart uint64 // block height when decay began; 0 if not set
}
```

**Validation:** If `DecayRate` is not positive, `DecayEpochStart` must be 0.

---

## 8. v7.0.0 Upgrade Values

| Parameter | Value | Notes |
|-----------|-------|-------|
| MaxSupply | 10^19 (100B CRO in basecro) | Hard cap |
| BurnedAddresses | Chain-specific burn address | `cro1qqq...` for mainnet |
| DecayRate | 0.068 (6.8%/month) | Monthly exponential decay |
| DecayEpochStart | Upgrade activation height | Decay begins at upgrade |
| Mint InflationMax | 1% | Lowered from default |
| Mint InflationMin | 1% | Fixed rate (no range) |
| Mint InflationRateChange | 0 | No dynamic adjustment |

## 9. Module Layout

```
x/inflation/
  module.go                 # AppModule, BeginBlocker registration
  abci.go                   # BeginBlocker: supply cap check
  client/cli/
    query.go, tx.go         # CLI commands
  keeper/
    keeper.go               # Keeper, params, decay epoch, cache
    mint.go                 # DeflationCalculationFn (decay formula)
    grpc_query.go           # Params query
    msg_server.go           # MsgUpdateParams handler
    genesis.go              # InitGenesis, ExportGenesis
  types/
    params.go               # Params validation
    genesis.go              # GenesisState validation
    keys.go                 # Store keys (ParamsKey, DecayEpochStartKey)
    expected_keepers.go     # BankKeeper, StakingKeeper interfaces
    *.pb.go                 # Generated protobuf types
```

**Expected keepers:** `BankKeeper` (GetSupply, GetBalance), `StakingKeeper` (BondDenom).

**Execution order:** Inflation BeginBlocker runs after Mint module so the supply cap is checked on the newly minted tokens.
